Pipeline Definition

The 6 Stages — Detailed Spec

Every application moves through these stages in order. Each stage has an assigned team, defined actions, and clear conditions to move forward or put on hold. Build your data model and CRM UI around this.

NEW APPLICATION
New Application
An application enters this stage the moment Razorpay confirms the ₹1,999 payment. Before payment, the lead exists in the system but is not a live application. After payment, a CRM record is created, a case ID is assigned, and it lands on the Screening Team's queue. The applicant is notified by push and SMS.

Once documents are cleared, the Project Report Team prepares a detailed project report (DPR) for the applicant. This is a formal document that banks and government bodies require — it outlines the business plan, projected cost, expected revenue, and loan utilisation plan. Integration Reference

NeoDove, Razorpay, Firebase — exact connection points

Three external services. Here is exactly what each one does in the system, what triggers them, and what data flows where.

NeoDove — Call Log Integration
NeoDove fires a POST webhook to your API at the end of every call: POST /api/webhooks/neodove. The payload includes: agent ID, contact phone number, call duration, call notes if entered, and timestamp. Your backend matches the phone number to the application in your database and writes a new activity log entry against that application. This is what populates the NeoDove Calls tab in the Application Detail screen. Never store the full audio — just the metadata and notes. NeoDove keeps the recordings on their end.
Razorpay — ₹1,999 Token Payment
Use the Razorpay React Native SDK. When the applicant hits "Pay ₹1,999", open the Razorpay payment sheet. On success, Razorpay fires POST /api/webhooks/razorpay to your backend. Verify the payment signature server-side (never trust the client callback alone). On verified success: create a live CRM application record, set stage to "New Application", trigger a Firebase push to the user, and send a confirmation SMS. If payment fails or is abandoned, the lead stays in "pending payment" state — do not create a CRM record.
Firebase Cloud Messaging — Push Notifications
Every stage change in the admin portal triggers a push notification to the applicant. Fire these notifications from your backend, never from the admin frontend. Events that trigger a push: payment confirmed, stage moved, document approved, document rejected (include reason text), application put on hold (include reason), loan sanctioned. Store the Firebase token against the user's profile on app registration. If the push fails (token expired), fall back to SMS via a provider like MSG91 or Twilio.
